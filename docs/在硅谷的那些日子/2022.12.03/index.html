Set​

概念​

  • 很像加强版的Map
  • 只有值，没有键，Set集合传入的参数为一维数组
  • 可以实现数组的去重

基本API​

new 关键字和 Set 构造函数可以创建一个空集合

  • const m = new Set();

基本操作​

  • add()增加值
  • has()查询
  • size 取得元素数量
  • delete()和 clear()删除元素

顺序和迭代​

  • 可以直接对集合实例使用扩展操作，把集合转换为数组
    • const s = new Set(["val1", "val2", "val3"]); console.log([...s]); // ["val1", "val2", "val3"]

      回调方式

    • const s = new Set(["val1", "val2", "val3"]); s.forEach((val, dupVal) => alert(`${val} -> ${dupVal}`)); // val1 -> val1 // val2 -> val2 // val3 -> val3`

实例​

  • 将set集合转换为真正的数组​

    • let arr1 = [...s1];*  *console.log(arr1); //[4, 5, 6, 7, 2, 3, 1]*  *console.log(arr1.sort()); //[1, 2, 3, 4, 5, 6, 7]

  • 将集合转换成数组​

    • 数组原型对象中的filter方法是过滤的方法,需要返回值，返回的是按照函数中条件进行过滤之后的元素形成的新数组
      • ES5


        • let newArr = [...s1].filter(function (item) {*  *//item表示数组中的每一项*  *// console.log(item);*  *//在s2这个集合中查找与每一个item项中相同的部分(has)*  *if (s2.has(item)) {*  *return true;*  *}*
      • ES6
        • let newArr = [...s1].filter(item => s2.has(item));*  *console.log(newArr);

Map​

复习​

  • ES6之前 用object存储键值对
    • 对象属性作为键，再使用属性来引用值

概念​

  • Map 是一种新的集合类型，为这门语言带来了真正的键/值存储机制。Map 的大多数特性都可以通过 Object 类型实现

基本api​

  • new 关键字和 Map 构造函数可以创建一个空映射
    • 创建的同时初始化实例，可以给 Map 构造函数传入一个可迭代对象，需要包含键/值对数组
      • 使用嵌套函数
      • const m1 = new Map([  ["key1", "val1"],  ["key2", "val2"],  ["key3", "val3"]
        alert(m1.size); // 3`


      • //使用自定义迭代器
        const m2 = new Map({  [Symbol.iterator]: function*() {  yield ["key1", "val1"];  yield ["key2", "val2"];  yield ["key3", "val3"];  } }); alert(m2.size);
  • set()方法添加键/值对
    • set()方法返回映射实例，因此可以把多个操作连缀起来，包括初始化声明：
    • const m = new Map().set("key1", "val1"); 
      m.set("key2", "val2")  
       .set("key3", "val3");
      alert(m.size);`
  • get()和 has()查询
  • size 属性获取映射中的键/值对的数量
  • 使用 delete()和 clear()删除值
  • 用for循环遍历

    • for (let item of m2) { // console.log(item); // }

与object进行对比

  • Object 只能使用数值、字符串或符号作为键
  • Map 可以使用任何 JavaScript 数据类型作为键

选择Object还是Map​

  • 内存占用​

    • Map 大约可以比 Object 多存储 50%的键/值对

  • 插入性能​

    • Map 的性能更佳

  • 查找速度​

    • 只包含少量键/值对，则 Object 有时候速度更快
    • 把 Object 当成数组使用，内存中使用更高效的布局
    • 如果代码涉及大量查找操作，选择 Object 更好

  • 删除性能​

    • Map 的 delete()操作都比插入和查找更快。如果代码涉及大量删除操作，选择 Map

继承​

ES5​

ES5没有支持面向对象的结构 但是用原型式继承模拟了同样的功能

构造函数功能​

区别​
  • 1.没有显示的创建对象
  • 2.属性和方法直接赋值给了 this
  • 3.没有 return

new的构造函数括号可加可不加 只要有 new 操作符，就可以调用相应的构造函数

创建构造函数，使用new操作符后发生以下步骤

  • 1.在内存中创建一个新对象
  • 2.新对象的prototype被赋值为新对象的prototype
  • 3.构造函数内部的this被赋值为这个新对象
  • 4.执行构造函数内部的代码，给新对象添加属性

console.log(person1.constructor == Person); // true

  • constructor 本来是用于标识对象类型的。但是认为 instanceof是更好的一种方式
  • 每个对象是Object的实例 也是Person的实例

没有使用new 调用Person的话 会直接加到window上 调用一个函数而没有明确设置 this 值的情况下（即没有作为对象的方法调用，或者没有使用 call()/apply()调用），this 始终指向 Global 对象

原型​

  • 之前用new 量大会导致内存占用过大 所以使用了原型
  • 原型链
    • 原型链为主要的继承方式
    • 原型指向构造函数，实例和原型之间就构造了一条原型链
    • 问题
      • 1.原型的所有值会在所有的实例中共享 所以一般会在构造函数中定义而不会定义在原型上，在使用原型继承上时，原型实际上变成了另一个类型的实例，实例属性变成了原型属性
      • 2.子类型在实例化时不能给父类型的构造函数传参

对象伪装（盗用构造函数）​

导入​
  • 为了解决原型包含引用值导致的继承问题

相当于所有的subtype中运行了Superytpe的代码

问题​
  • 必须在构造函数中定义方法，因此函数不能重用。
  • 子类也不能访问父类原型上定义的方法，因此所有类型只能使用构造函数模式。

组合继承​

  • 导入：综合原型链和盗用构造函数
  • 组合继承弥补了原型链和盗用构造函数的不足，是 JavaScript 中使用最多的继承模式

ES6​

开始正式的支持类和继承，其实也就是封装了ES5的原型继承的语法糖

类​

概念:实际还是用的原型和构造函数

定义​
  • 类声明
    • class Person {}
  • 类表达式
    • const Animal = class {};
对比​

函数声明可以提升，但是类不能提升

函数受函数作用域限制，类受块作用域限制

主要区别​
  • 调用类构造函数必须用new操作符
  • 普通构造函数如果不使用new调用 那么就会以全局的this作为内部对象

类构造函数​

constructor​

  • 在类定义块内部创建类的构造函数
  • 告诉new创建类的新实例时 应该调用这个函数

实例化​

  • 使用new实例化后，会执行
    • 1.在内存中创建一个新对象
    • 2.新对象的prototype被赋值为新对象的prototype
    • 3.构造函数内部的this被赋值为这个新对象
      1. 执行构造函数内部的代码
    • 5.如果构造函数返回非空对象，则返回该对象；否则，返回刚创建的新对象
实例成员​
  • 每个实例都对应一个唯一的成员对象
  • 所有成员不会再原型上共享
原型方法​
  • 在块中定义的语法成为原型方法
  • 不能给块中的原型添加原始值和对象
支持获取和设置访问器​
  • set
  • get

静态类方法​

  • 静态成员每个类上只能有一个
  • static 关键字作为前缀，this指向类本身

继承​

概念​
  • 支持单继承，使用 extends 关键字
    • 继承任何拥有[[Construct]]和原型的对象
    • let Bar = class extends Foo {}
构造函数​
super​
  • 类构造函数中使用 super 可以调用父类构造函数。
  • 可以通过super关键字引用它们的原型
  • 注意事项
    • 1.super必须在构造函数中的第一句话
    • 2.super 只能在派生类构造函数和静态方法中使用。
    • 3.不能单独引用 super 关键字，要么用它调用构造函数，要么用它引用静态方法
    • 4.调用 super()会调用父类构造函数，并将返回的实例赋值给 this
注意事项​
  • 情况1：当如果子类继承了父类之后，则默认会执行父类的构造器,这种情况是在子类没有明显写出自己的构造器的时候
  • 情况2：当如果子类已经明显了写出了自己的构造器，则实例化子类的对象的时候会优先执行自己的构造器
  • 但是在自己的构造器的内部必须强制调用父类的构造

分类​

  • 接口继承
    • 继承方法签名
  • 实现继承
    • 继承实际的方法

拷贝（克隆）​

浅拷贝​

概念​

  • 当无论是新对象还是老对象，修改哪一个对象的属性值，都会印象另一个对象的元素，因为修改的是对象的内存地址
  • 浅拷贝会经常发生在数组/对象身上

方法​

1、直接赋值（地址值） 无论值是什么类型，一个改了另一个也变​
1.1 数组​
  • let arr = [1, 2, 3];  //0x100*` 
    let arr1 = arr; //将arr的内存地址值赋值到了arr1身上  0x100 //修改对象中的元素值         
    arr1[0] = 10;         
    arr.push(4);         
    console.log(arr);        
    console.log(arr1);
1.2 对象​
  • `*let obj = { a: 1, b: 2 }; //0x18*` 
    `*let obj1 = obj; //0x18*`  `*//修改元素值*` 
    `*obj.a = 3;*`  
    `*obj.c = 4;*`  
    `*console.log(obj);*`  
    `*console.log(obj1);*`
2、通过数组中的方法​
  • concat和slice 当如果元素值为基本数据类型，则哪一个改了哪一个变 当如果元素值为引用数据类型，则一个改了另一个也变
    concat(合并)​

    • 基本数据类型

      • ***let arr = [1, 2, 3, 4];***

      • let arr1 = arr.concat();

        • 当concat方法没有参数的话，相当于是把arr这个数组中的元素一个一个的复制到了arr1里面
        • 所以arr1相当于arr的副本

    • 引用数据类型

      - 
      - > `*let arr2 = [1, 2, 3, { name: '张三' }];*`  `*let arr3 = arr2.concat();*`  `*//修改*`  `*arr2[3].name = '李四'*`  `*console.log(arr2);*`  `*console.log(arr3);*`
      slice(截取)​

    • let arr4 = [1, 2, 3, 4, 5]

    • 当slice方法如果没有添加参数的话，则默认为0，相当于直接从数组的第一位元素开始截取到最后一位

    • let arr5 = arr4.slice();

    • 修改

      • ***arr4[0] = 10***
3、扩展运算符​
  • 当如果元素值为基本数据类型，则哪一个改了哪一个变
  • 当如果元素值为引用数据类型，则一个改了另一个也变

  • let arr6 = [1, 2, 3, { name: '张三' }]; let arr7 = [...arr6]; //修改 arr6[0] = 10; arr7[3].name = '李四' console.log(arr6); console.log(arr7);

4、Object.assign​
  • 当如果元素值为基本数据类型，则哪一个改了哪一个变
  • 当如果元素值为引用数据类型，则一个改了另一个也变

  • let obj = { name: '于谦', age: 50, hobby: ['抽烟', '喝酒', '烫头'] } //将一个对象中的键值对复制到另一个对象中，就可以使用Object.assign let obj1 = Object.assign({}, obj); //修改 obj1.age = 55; obj1.hobby.shift(); console.log(obj); console.log(obj1)

深拷贝​

概念​

  • 深拷贝：无论修改的是新对象还是原对象，都只是被修改的那个对象中的元素值有变化，而不会影响到另一个对象

方法​

1、JSON.parse(JSON.stringfy)​
  • JSON.stringify:将一个JSON对象转换成JSON字符串 注意这种JSON对象的深拷贝形式，不能直接复制函数类型的值！！！
  • JSON.parse:将一个JSON字符串转换成JSON对象
2、for..in 递归遍历​
3.lodash中的cloneDeep()​

手写深拷贝​

区分（从堆栈来说）​

  • 浅拷贝
    • 在拷贝数据时，只把栈中的数据拷贝方式就是浅拷贝，修改拷贝出来的数据可能会影响原数据
  • 深拷贝
    • 深拷贝是连同堆中的数据一起完全复制，修改新数据，不会影响原数据
