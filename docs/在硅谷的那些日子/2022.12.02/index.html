symbol

概念​

更像是一种唯一性标识，主要针对于对象的键名在整体对象中是一种唯一的,表示独一无二的值

数据类型：​

    ES5: number、string、boolean、null、undefined、array、object、function

    ES6: symbol

定义​

let s1 = Symbol();
console.log(s1);

参数更多的是基本数据类型(number/string/boolean)

特点​

使多个symbol的值是一致的，也不能说他们是相等的，因为每一个symbol都是绝对唯一的symbol

let s5 = Symbol('js');
let s6 = Symbol('js');
console.log(s5 === s6); //false

Symbol和其他数据做运算

迭代器

概念​

实现 Iterable 接口的数据结构都可以被实现 Iterator 接口的结构“消费”（consume）。迭代器（iterator）是按需创建的一次性对象。每个迭代器都会关联一个可迭代对象，而迭代器会暴露迭代其关联可迭代对象的 API

以下常见的几种数据结构中包含了迭代器​

集合(NodeList、HTMLCollection、Arguments、Set、Map)

字符串对象(String)

数组(Array)

工作原理​

  • 创建一个指针对象，指向当前数据结构的起始位置
  • 第一次调用对象的next方法，指针自动指向数据结构的第一个成员
  • 接下来不断调用next方法，指针一直向后移动，直到指向最后一个成员
  • 每调用next方法返回一个包含value和done属性的对象
    • value表示循环的值
    • done的值默认为false，当迭代完毕时就会显示为true

接收可迭代对象的原生语言特性​

 for-of 循环

 数组解构

 扩展操作符

 Array.from()

 创建集合          

 创建映射

 Promise.all()接收由期约组成的可迭代对象                       

 Promise.race()接收由期约组成的可迭代对象

迭代器协议​

迭代器 API 使用 next()方法在可迭代对象中遍历数据。每次成功调用 next()，都会返回一个 IteratorResult 对象，其中包含迭代器返回的下一个值。若不调用 next()，则无法知道迭代器的当前位置

next

返回的迭代器对象 IteratorResult

done 是一个布尔值，表示是否还可以再次调用 next()取得下一个值

value 包含可迭代对象的下一个值（done 为false），或者 undefined（done 为 true）。

注意：迭代器维护着一个指向可迭代对象的引用，因此迭代器会阻止垃圾回收程序回收可迭代对象！！！

利用for of循环迭代器​

调用函数​

迭代器其实也是一个指针对象

console.log(arr[Symbol.iterator]()); //Array Iterator {}

//而在这个指针对象的原型里面有一个next()方法,可以将数组中的元素进行逐一循环

调用next()方法​

value:数组中的元素

done：表示迭代是否完毕，默认为false，如果迭代完毕了为true

let obj = arr[Symbol.iterator]();
        console.log(obj.next()); //{value: 10, done: false}
        console.log(obj.next()); //{value: 20, done: false}
        console.log(obj.next()); //{value: 30, done: false}
        console.log(obj.next()); //{value: undefined, done: true}

for (let item of arr) {

         console.log(item);
     }

如果在for of中还想要获取下标的话，则需要借助于数组原型身上的一个方法：entries()

for (let [index, item] of arr.entries()) {
             console.log('下标:', index);
             console.log('值:', item);}

for of与 for in的对比

for of和for in的相同点和不同点：

相同点：​

都是用来遍历数据的

不同点：​

1）for of是ES6中提出的新的遍历方式，只要是数据中有iterator接口，才可以进行for of

2）如果for of 遍历数组，item则是数组中的每一个元素

     如果for in遍历数组，item则是数组的每个元素的下标

3）for of 不能遍历对象，但是for in可以遍历对象

4）for of 本身是无法获取数组下标的，但是可以借助于entries()方法
