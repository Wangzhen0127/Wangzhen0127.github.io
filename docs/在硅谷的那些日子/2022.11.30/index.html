ES5

复习​

call、apply、bind​

相同点

call和apply来说：主要是函数中传递的实际参数上的形式不同

           call实参是一个一个的传递

           apply实参要求必须是一个数组

不同点

call和bind来说：主要在于修改完函数的this指向之后，函数是否会立即执行的问题

           call先修改this指向，然后再执行函数体

           bind先修改this指向，然后不会立即执行函数体，这个方法是有返回值的，

           返回的是与原函数结构一模一样的新函数结构(地址发生了变化)

ES6

let​

与var对比​

let 声明的范围是块作用域，而 var 声明的范围是函数作用域。

let 声明的变量不会在作用域中被提升

let 声明之前的执行瞬间被称为“暂时性死区”

let 在全局作用域中声明的变量不会成为 window 对象的属性

const​

和let基本相同，差别声明变量时必须同时初始化变量

修改 const 声明的变量会导致运行时错误

解构赋值​

使用与对象匹配的结构来实现对象属性赋值

解构赋值不一定与对象的属性匹配。赋值的时候可以忽略某些属性，而如果引用的属性不存在，则该变量的值就是 undefined

可以在解构赋值的同时定义默认值

例子​

数组

语法：let [变量1,变量2...] = 数组

对象

对象的属性没有次序，变量必须与属性同名，才能取到正确的值
